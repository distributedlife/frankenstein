Typical usage

The application under test is likely to be launched by a shell script or batch file. In order to integrate Frankenstein, you'll need to locate the line in the script that launches the application under test. This will typically look like this
java ... com.your.package.Application
Change the line so that it looks like this:
java ... com.thoughtworks.frankenstein.application.PipingMain com.your.package.Application

Customization

If you're extending Frankenstein, you'll need to write your own main class.
package com.your.package;

public class FrankensteinWrapper {
public static void main(String[] args) {
FrankensteinIntegration integration = new FrankensteinIntegration(YourMainClass.class);
integration.registerEvent(YourEvent.class); //Register events, if required
integration.registerRecorder(YourRecorder.class); //Register recorder, if required
integration.start(args); //Finally, get Frankenstein to launch your application.
}
}
Your application's launch line will now need to look like this:
java ... com.your.package.FrankensteinWrapper
